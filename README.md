# API 管理系统

[![Version](https://img.shields.io/badge/version-1.0.0-blue.svg)](https://github.com)
[![PHP](https://img.shields.io/badge/PHP-%3E%3D7.0-777BB4.svg)](https://www.php.net/)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)

## 功能特性

### 前台功能

| 功能 | 说明 |
|------|------|
| 站点首页 | 展示站点介绍、最新公告、友情链接、访问统计 |
| API 列表 | 浏览所有接口，支持按名称/描述搜索 |
| API 详情 | 查看接口地址、参数说明、请求/响应示例 |
| 用户注册/登录 | 支持用户自主注册，首个注册用户自动成为管理员 |
| 个人中心 | 修改昵称、密码，查看/复制/重置 API 密钥 |

| 响应式布局 | 适配 PC 与移动端，侧边栏可折叠 |

### 后台功能

| 功能 | 说明 |
|------|------|
| 数据面板 | 用户总数、访问次数、API 数量、总调用次数统计 |
| API 管理 | 添加、编辑、删除接口，配置参数与示例 |
| 用户管理 | 查看用户列表，重置用户密码 |
| 站点设置 | 站点名称、介绍、公告、弹窗、音频、友情链接、备案号等 |

## 快速开始

### 1. 克隆项目

```bash
git clone https://github.com/你的用户名/api2.0.git
cd api2.0
```

### 2. 部署到 Web 服务器

将项目文件放置到网站根目录（或虚拟主机目录），例如：

- **Apache / Nginx**：`/var/www/html/api2.0/`
- **宝塔面板**：创建站点后上传至网站目录
- **本地开发**：使用 PHP 内置服务器

```bash
php -S localhost:8080
```

浏览器访问：`http://localhost:8080`

### 3. 首次使用

1. 打开站点首页，点击 **注册**
2. **第一个注册的用户将自动成为管理员**
3. 登录后进入 **后台管理**，完成站点设置并添加 API 接口

> 首次访问时，系统会自动在 `data/` 目录下生成默认配置文件。

## 使用指南

### 前台用户

#### 注册与登录

1. 访问 `register.php` 填写昵称、账号、密码（密码至少 6 位）
2. 注册成功后自动登录，或前往 `login.php` 手动登录

#### 浏览 API

1. 进入 **API 列表** 页面
2. 可使用搜索框按接口名称或描述筛选
3. 点击 **查看详情** 进入接口文档页，查看：
   - 接口地址
   - 请求参数（名称、类型、是否必填、描述）
   - 请求示例 / 响应示例
   - 接口启用/停用状态

> 未登录用户也可浏览 API 列表与详情；登录用户查看详情时会记录调用次数。

#### 个人中心

登录后进入 **个人中心**，可进行以下操作：

- **修改昵称**：更新显示名称
- **修改密码**：需验证原密码
- **API 密钥**：查看、复制或重新生成专属 API Key（用于后续接口鉴权扩展）

---

### 后台管理员

管理员登录后，侧边栏会出现 **后台管理** 入口，或直接访问 `admin/index.php`。

#### 数据面板

查看系统运行概况：

- 用户总数
- 总访问次数
- API 数量
- 总调用次数
- 服务器环境信息（操作系统、PHP 版本等）

#### API 管理

路径：`admin/apis.php`

**添加接口：**

| 字段 | 说明 |
|------|------|
| 接口名称 | 显示在列表中的名称 |
| 接口地址 | 实际 API URL |
| 接口描述 | 简要说明接口用途 |
| 请求参数 | 可动态添加多个参数（名称、类型、是否必填、描述） |
| 请求示例 | 示例请求代码或 URL |
| 响应示例 | 示例返回 JSON |
| 接口状态 | 启用 / 停用 |

**其他操作：**

- 点击 **编辑** 修改已有接口
- 点击 **删除** 移除接口（需确认）
- 使用搜索框快速查找接口

#### 用户管理

路径：`admin/users.php`

- 查看所有注册用户（昵称、账号、角色、API 调用次数）
- 支持搜索用户
- 可重置任意用户的密码

#### 站点设置

路径：`admin/settings.php`

| 设置项 | 说明 |
|--------|------|
| 站点名称 | 显示在浏览器标题和导航栏 |
| 站点介绍 | 首页「站点介绍」区域内容，支持换行 |
| 站点公告 | 首页「最新公告」区域内容 |
| 弹窗提醒内容 | 用户首次进入首页时弹出的提示（留空则不显示） |
| 音频链接 | 弹窗关闭后播放的音频 URL（可选） |
| 友情链接 | 可动态添加/删除多个友情链接 |
| 备案号 | 页脚 ICP 备案信息 |
| 版权所有者 | 页脚版权文字 |

## 数据存储说明

系统采用 JSON 文件存储，所有数据位于 `data/` 目录：

### users.json — 用户数据

```json
{
  "id": "唯一ID",
  "nickname": "昵称",
  "account": "登录账号",
  "password": "MD5加密密码",
  "role": "admin | user",
  "api_key": "用户API密钥",
  "created_at": "注册时间",
  "api_calls": []
}
```

### apis.json — API 数据

```json
{
  "id": "唯一ID",
  "name": "接口名称",
  "url": "接口地址",
  "description": "接口描述",
  "parameters": [
    {
      "name": "参数名",
      "type": "string | integer | boolean | array",
      "required": true,
      "description": "参数说明"
    }
  ],
  "request_example": "请求示例",
  "response_example": "响应示例",
  "status": "active | inactive",
  "call_count": 0
}
```

### settings.json — 站点设置

包含站点名称、介绍、公告、弹窗、音频、友情链接、备案号、版权等配置。

### visit_log.json — 访问日志

记录每次页面访问的 IP、时间、User-Agent，用于首页访问统计。

## 常见问题

**Q：忘记管理员密码怎么办？**

可直接编辑 `data/users.json`，将对应用户的 `password` 字段改为新密码的 MD5 值，或删除该文件后重新注册（会丢失所有用户数据）。

**Q：如何备份数据？**

复制整个 `data/` 目录即可，其中包含全部用户、API 和站点配置。

**Q：如何迁移到新服务器？**

将整个项目目录（含 `data/`）上传到新服务器，确保 Web 服务器指向项目目录并赋予 `data/` 读写权限。

**Q：API 列表为空？**

需使用管理员账号登录后台，在 **API 管理** 中添加接口。

**Q：第二个用户能当管理员吗？**

当前版本中，仅 **第一个注册用户** 自动获得管理员权限，后续用户均为普通用户。

## 注意事项

1. **安全建议**：本系统使用 MD5 存储密码，适用于内网或演示环境。生产环境建议升级为 `password_hash()` 等更安全的方案，并启用 HTTPS。
2. **目录权限**：确保 `data/` 和 `uploads/` 对 Web 服务器进程可写，否则无法保存数据。
3. **数据目录保护**：建议在 Web 服务器配置中禁止直接访问 `data/` 目录，防止 JSON 数据被下载。
4. **访问日志**：`visit_log.json` 会随访问增长，长期使用建议定期清理或改为轮转策略。
5. **API 密钥**：用户在个人中心生成的 API Key 可用于后续扩展接口鉴权，当前版本主要用于展示与管理。
